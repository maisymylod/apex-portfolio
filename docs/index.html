Overview

A $1,000 paper portfolio run by an autonomous agent.

The portfolio holds 13 names inferred from Situational Awareness Capital's May 18 2026 13F and the situational-awareness.ai essays (transformative AI by roughly 2027, power generation as the binding constraint). There is no real broker and no real money. Positions are sized by thesis conviction.

Constraint 1 Mirror only. No positions beyond the published thesis and filing.
Constraint 2 Marked to market every weekday. Reproducible, with no manual adjustment after the run.

A GitHub Action runs each weekday at 4:30pm ET. It pulls prices, marks the book to market, computes realized and SPY-relative analytics, updates a per-ticker conviction bias, and writes a journal entry. Three notes on how the analytics are reported:

  1. Returns are reported against SPY. Absolute P&L is shown alongside active return, beta, alpha, tracking error and information ratio, since absolute return alone is not informative for a portfolio this young.
  2. Simulated metrics are labelled. The Sharpe and VaR figures from the 10,000-path Monte Carlo are model output, not a realized track record, and are marked SIMULATED wherever they appear.
  3. The pipeline was audited for data-integrity bugs. One example is below.
DISCOVERY.md — Gap G : silent data-quality failure RESOLVED

When a ticker's price failed to download, the original pipeline did not raise an error. It dropped that position from the total, so its market value disappeared and the resulting drop was recorded in the history file as a real loss.

A later audit pass (DISCOVERY.md, Gap G) identified this. The current version marks the affected day degraded rather than silently rewriting the book. The same audit found the learning loop's docstring describing SPY-relative math that the code did not implement.

Open questions

  • Whether model intelligence provides a measurable edge in liquid markets.
  • How much reported AI-trading performance reflects similar lookahead or data-integrity errors.
  • How the conviction-bias loop should be validated against realized outcomes.
